ABOUT ME

Hi! I'm Oskari Mantere, a Finnish second year Applied Math student living in Helsinki. I enjoy coffee, sushi and all things technology.

I have had plenty of experience with computers since my teenage years, and I have always had a healthy appetite for learning new things.

In addition to having the privilege of studying in one of the top universities in Finland, I have also been fortunate enough to have had some experience working at two interesting companies. Please feel free to read about my thoughts on my time at Netmedi in a post on their blog.

Netmedi

This summer I worked at Netmedi on an internal content management system tool for the Kaiku service and the Netmedi platform. The content management system is a Rails application with a Postgres database and a React frontend. The existing data was in the form of YAML documents, which were prone to editing errors, and required tedious manual work both to edit and to deploy. I migrated the existing data into the CMS and designed and developed an user friendly interface for the data, as well as a protocol and endpoints for uploading the data into running deployments without the need of a re-deploy.

Visit site

Hobbydeed

In the winter of 2015-2016 I was involved in Mikko Hedman's project Hobbydeed, a web service aiming to create a toolbox for all kinds of groups to get organized. I worked across the stack, both client- and serverside, server administration and deployments. During my time I did lots of improvements to the UI and implemented several new features, such as Facebook login integration, intelligent group search with typeahead and various group data views. I also did refactorings to the backend as well as the UI.
The technology stack consisted mostly of AngularJS complemented by SASS and the Twitter Bootstrap framework in the front-end, and a Java EE/Spring based backend using an embedded Neo4J graph database.

Timio

Timio is a web service to track how you spend your time on all of your devices. Started this one because a lot of people, myself included, spend a lot of time on technology, and thought it would be nice to make this tool for visualizing the app usage data on all devices at once. Also an excuse for diving a bit into the APIs on different platforms. :) Still a work in progress! So far I'm working on clients for Mac, Linux and Android.

Aalto WebOodi Calendar Tool

At the time I arrived at Aalto University, the course system at had no way to export timetables to any external calendar, and the built-in system sucked. So i made this little tool that allows exporting of everything to Google Calendar.
The functionality is now provided by MyCourses, so I don't suggest anyone to actually use this anymore. However, it was a fun project that I'm happy about, which is why I'll keep it hosted. :)
